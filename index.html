Build a Portfolio Site

This is my 1st attempt at creating a responsive web page from scratch. This project is submitted to Udacity and graded. Combining lessons from responsive design (see Featured Work #3) and lessons from responsive images (Featured Work #2), I was able to complete this project.

Responsive design allows my web page to be viewed on devices of different screen sizes, and even on different viewport sizes of browsers on computers that allow browsers to be resized. Responsive images directs the browser to download the correct images for the various screen sizes. This also forms part of the effort to optimize the web page --- significant amount of bandwidth is saved by downloading (almost) exactly the right-sized images, rather than downloading a huge image viewable on large computer screens and pointlessly overkill on smaller mobile screens.

Responsive Images

This project (ungraded) covers concepts for responsive images. Though the project is ungraded, I made some much needed corrections and improvements to Udacity's official solution.

Udacity's official solution does not perform cropping (via grunt-responsive-images) properly because grunt-responsive-images has a crop bug. See my correction at: https://github.com/andismith/grunt-responsive-images/pull/124. Eventually, because grunt-responsive-images does not adequately expose the full power of the underlying GraphicsMagick it uses, I had to fall back on invoking GraphicsMagick directly (via grunt-exec).
